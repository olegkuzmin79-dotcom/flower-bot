# Хранение данных и бэкапы (бесплатно)

## Почему пропадали данные

База `data/bot.db` лежала **внутри контейнера**. Каждый `git push` → новый контейнер → пустая база.

Код в GitHub не затирается. Затирается только файл на диске Railway.

## Шаг 1 — постоянный диск (обязательно)

1. [Railway](https://railway.app) → проект **flower-bot** → сервис бота  
2. **Volumes** → **Add Volume**  
3. Mount path: `/app/data`  
4. Redeploy  

После этого `bot.db` и `data/backups/` переживают деплои.

> На бесплатном Hobby volume обычно доступен. Если кнопки нет — напиши в поддержку Railway или используй только бэкапы в Telegram (шаг 2).

## Шаг 2 — веб-админка

На том же сервисе поднимается Flask:

| URL | Описание |
|-----|----------|
| `https://<твой-railway-url>/admin` | Клиенты, праздники, заказы |
| `/admin/export/orders.csv` | Экспорт в CSV → Google Sheets |

**Variables в Railway:**

| Переменная | Значение |
|------------|----------|
| `WEB_ADMIN_PASSWORD` | Длинный пароль только для тебя |
| `WEB_SECRET_KEY` | Случайная строка 32+ символов |
| `ADMIN_WEB_ENABLED` | `1` |

В **Settings → Networking** включи **Public Networking** — Railway выставит `PORT` сам.

## Шаг 3 — бэкапы в Telegram

Каждый день в **04:00 МСК** бот присылает в `ADMIN_CHAT_ID` (твой Telegram):

1. Файл `bot_YYYYMMDD_HHMM.db` — полная база  
2. CSV: клиенты, праздники, заказы — можно импортировать в Google Sheets  

Локальные копии: `data/backups/` (последние 14 по умолчанию).

Ручной бэкап: команда `/backup` в боте (только с твоего chat id).

## Google Sheets (бесплатно)

1. Админка → **CSV заказы** (или клиенты)  
2. Google Sheets → **Файл → Импорт → Загрузка**  
3. Разделитель: запятая, кодировка UTF-8  

Или дождись ежедневных CSV в Telegram.

## Что где хранится

| Что | Где |
|-----|-----|
| Живые данные | `/app/data/bot.db` (volume) |
| Копии на диске | `/app/data/backups/` |
| Архив в облаке | Telegram (файлы у тебя в чате) |
| Код бота | GitHub `flower-bot` |

## Восстановление из бэкапа

1. Скачай `bot_*.db` из Telegram  
2. Railway → volume или замени файл через одноразовый deploy с загрузкой (проще: попроси помочь при первом разе)  
3. Либо локально положи в `data/bot.db` и задеплой с volume  

---

**Итого:** volume = данные не пропадают при деплое; админка = удобный просмотр; Telegram + CSV = страховка бесплатно.
